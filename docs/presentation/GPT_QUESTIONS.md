# GPT에게 물어볼 질문 리스트

## 1. AIhub 데이터 구성 및 활용 방안

### 질문
```
AIhub의 축구 전술 데이터셋(데이터셋 ID: 71482)의 구체적인 구성이 어떻게 되어 있나요?

1. 데이터 타입: 영상(MP4)? 이미지? 이벤트 로그? 메타데이터?
2. 데이터 스키마: 어떤 필드들이 포함되어 있나요?
3. 라벨 정보: 수동 라벨링된 전술 패턴(압박, 옵사이드, 공격 전환 등)이 포함되어 있나요?
4. 데이터 규모: 경기 수, 영상 길이, 이벤트 수 등
5. 활용 방안: 현재 Track2 이벤트 로그만으로는 부족한 정보(옵사이드 라인, 압박 강도, 선수 위치 등)를 AIhub 데이터로 보완할 수 있나요?
```

### 목적
- AIhub 데이터의 실제 구성 파악
- 현재 모델의 한계(이벤트 로그만)를 보완할 수 있는지 확인
- 데이터 통합 가능성 평가

## 2. 추가 무료 축구 데이터셋 조사

### 질문
```
현재 프로젝트는 Track2 이벤트 로그만 사용 중입니다. 다음 정보를 보완할 수 있는 무료/공개 축구 데이터셋을 추천해주세요:

1. **옵사이드 라인 정보**: 선수 위치와 옵사이드 라인 판정 데이터
2. **압박 강도**: 공을 가진 선수 주변의 압박 상황 데이터
3. **선수 위치 추적**: 프레임별 선수 좌표 데이터
4. **공 위치 추적**: 프레임별 공 좌표 데이터
5. **전술 메타데이터**: 포메이션, 전술 패턴 라벨 등

각 데이터셋의:
- 출처 및 라이선스
- 데이터 형식 및 스키마
- Track2 이벤트 로그와의 통합 가능성
- 수집/전처리 방법
```

### 목적
- 이벤트 로그의 한계 보완
- 더 정확한 전술 분석을 위한 데이터 확보
- 모델 성능 개선을 위한 피처 확장

## 3. 현재 모델의 한계 및 보완 방안

### 질문
```
현재 `will_have_shot` 모델은 Track2 이벤트 로그만 사용합니다. 하지만 실제 경기 분석에는 다음 정보가 필요합니다:

1. **옵사이드 라인**: 공격 전환 시 옵사이드 활용 여부
2. **압박 강도**: 공을 가진 선수 주변의 압박 상황
3. **선수 간 거리**: 공격/수비 선수 간 거리 및 포메이션
4. **공간 점유**: 페널티 박스 내 공간 점유율

이벤트 로그만으로는 위 정보를 추론하기 어렵습니다. 다음을 제안해주세요:

1. 이벤트 로그에서 간접적으로 추론 가능한 피처 (예: 최근 패스 방향으로 압박 추정)
2. 외부 데이터(영상, 위치 추적) 없이 개선할 수 있는 방법
3. 외부 데이터가 필요한 경우, 최소한의 데이터로 보완하는 방법
```

### 목적
- 현재 모델의 한계 명확화
- 이벤트 로그만으로 개선 가능한 방안 도출
- 외부 데이터 필요성 판단

## 4. 전술 분석을 위한 최소 데이터 요구사항

### 질문
```
프로 수준의 전술 분석을 위해서는 다음 정보가 필요합니다:

1. **옵사이드 판정**: 선수 위치 + 공 위치 + 옵사이드 라인
2. **압박 강도**: 공 주변 선수 수, 거리, 속도
3. **공간 점유**: 페널티 박스 내 공격/수비 선수 비율
4. **전술 패턴**: 포메이션, 전환 속도, 공격 방향

현재 Track2 이벤트 로그만으로는 위 정보 중 어느 정도를 추론할 수 있나요?
그리고 각 정보를 얻기 위한 최소한의 추가 데이터는 무엇인가요?

예를 들어:
- 옵사이드: 선수 위치 데이터 필요 (이벤트 로그로는 불가)
- 압박: 이벤트 로그의 패스/드리블 타입으로 간접 추정 가능?
- 공간 점유: 이벤트 로그의 좌표 정보로 부분적 추정 가능?
```

### 목적
- 데이터 요구사항 명확화
- 우선순위 설정 (어떤 데이터가 가장 중요?)
- 실현 가능한 개선 방안 도출

## 5. 모델 성능 개선을 위한 데이터 전략

### 질문
```
현재 모델 성능:
- PR-AUC: 0.0829 (목표: 0.12+)
- Precision@10: 0.20 (목표: 0.30+)
- Precision: 0.0909 (목표: 0.20+)

이벤트 로그만으로는 한계가 있습니다. 다음 중 어떤 전략이 가장 효과적일까요?

1. **이벤트 로그 고도화**: 더 정교한 피처 엔지니어링 (현재 54개 피처)
2. **외부 데이터 통합**: AIhub 영상 데이터 → 이벤트 로그 변환 파이프라인
3. **하이브리드 접근**: 이벤트 로그 + 최소한의 위치 데이터 (선수/공 좌표만)
4. **전이 학습**: 다른 축구 데이터셋으로 사전 학습 후 미세 조정

각 전략의:
- 예상 성능 개선 폭
- 구현 난이도 및 비용
- 실현 가능성 (데이터 가용성)
```

### 목적
- 최적의 개선 전략 선택
- 리소스 효율적인 접근 방법 도출
- 현실적인 목표 설정

## 6. AIhub 데이터 통합 로드맵

### 질문
```
AIhub 데이터를 활용하여 현재 모델을 개선하려면:

1. **데이터 변환 파이프라인**: 영상/이미지 → 이벤트 로그 변환 방법
2. **라벨 활용**: AIhub의 수동 라벨링된 전술 패턴을 학습 데이터로 활용
3. **피처 확장**: 영상에서 추출한 정보(압박, 옵사이드 등)를 피처로 추가
4. **앙상블**: 이벤트 로그 기반 모델 + 영상 기반 모델 결합

각 단계의:
- 기술적 요구사항 (CV 모델, 추적 알고리즘 등)
- 구현 난이도
- 예상 성능 개선
- 우선순위 (어떤 것부터?)
```

### 목적
- AIhub 데이터 활용 계획 수립
- 단계별 구현 로드맵 작성
- 리소스 배분 결정
