- **파일 수**: 89,158개

### 1-3. 라벨 정보
- **전술 상황 분류**:

- **팀/개인 스탯 분류**: 포지션/상황 분류

### 1-4. 데이터 스키마 (예상)

- **예상 구조**:
  - (A) 클립 단위: `{clip_id, match_id, start_ts, end_ts, tactical_label, referee_label, ...}`
  - (B) 프레임/객체 단위: `{frame_idx: [{track_id, class, bbox, team_id, role, ...}], ...}`

### 1-5. Track2 보완 가능성
- **조건부 가능**: 추적/좌표 데이터가 있으면 오프사이드 라인, 압박 강도 계산 가능
- **제한사항**: 5FPS로 인한 정밀도 제한, 전술 라벨만 있으면 제한적

## 2. 무료/공개 데이터셋 추천

### 2-1. 추적(선수/공 좌표) 포함 데이터셋

#### (1) SkillCorner OpenData
- **내용**: 선수/공 추적 + 이벤트 + 공격/수비 세그먼트
- **형식**: 프레임별 좌표
- **출처**: [GitHub](https://github.com/SkillCorner/opendata)
- **통합**: Track2와 event_type/time/x,y 레벨 매핑 가능

#### (2) Metrica Sports Sample Data
- **내용**: 교육/연구용 추적+이벤트 샘플
- **용도**: 피처 설계/검증(pressure, spacing, speed) 레시피
- **출처**: [GitHub](https://github.com/metrica-sports/sample-data)

#### (3) Bundesliga 통합 데이터 (7경기)
- **내용**: 이벤트 + 포지션(추적) + 메타
- **용도**: 추적 기반 파생 피처 검증
- **출처**: [Nature](https://www.nature.com/articles/s41597-025-04505-y)

#### (4) SoccerTrack v2
- **내용**: 파노라마 비디오 + 2D 피치 좌표 라벨
- **용도**: 영상→좌표 복원/추적→전술 피처 파이프라인 R&D
- **출처**: [arXiv](https://arxiv.org/pdf/2508.01802)

#### (5) SoccerNet Tracking
- **내용**: 방송 카메라 기반 트래킹 벤치마크 (12경기)
- **용도**: 영상에서 추적 추출 검증
- **출처**: [GitHub](https://github.com/SoccerNet/sn-tracking)

### 2-2. 이벤트 데이터 중심 (추적 없음/부분)

#### (6) StatsBomb Open Data
- **강점**: 
  - 공개 이벤트 데이터 풍부
  - **Pressure(압박) 이벤트 포함** (`under_pressure`, `counterpress`)
  - 오프사이드 이벤트 포함
- **한계**: 오프사이드 "라인"은 추적 필요
- **출처**: [GitHub](https://github.com/statsbomb/open-data)

#### (7) Wyscout Soccer Match Event Dataset
- **강점**: 대규모 spatio-temporal event 데이터 (CC BY 4.0)
- **용도**: Track2와 유사, 전이학습/사전학습 적합
- **출처**: [Nature](https://www.nature.com/articles/s41597-019-0247-7)

## 3. Track2 이벤트 로그만으로 가능한 보완

### 3-1. 본질적으로 어려운 것
- **오프사이드 라인**: 최종 수비수 x좌표 필요 → 이벤트 로그만으로 불가
- **압박 강도(진짜)**: 공 주변 수비수 거리/속도 필요
- **선수 간 거리/포메이션/공간 점유**: 추적 없이는 정확한 계산 불가

### 3-2. 간접 추론 가능한 피처

1. **전개 속도/직진성 (Directness)**
   - 최근 N초 `Δx / Δt` (전진 속도)
   - 전진 패스 비율
   - 중앙/측면 전환 빈도

2. **공 소유/흐름 (포제션 체인)**
   - 체인 길이(이벤트 수)
   - 체인 시간
   - 체인 내 성공률
   - 마지막 3이벤트 타입 패턴

3. **위험 지역 진입 (Zone entry)**
   - 파이널 서드 진입
   - 박스 근처 진입
   - 하프스페이스 진입

4. **압박 프록시 (Proxy)**
   - 턴오버 직후 5초 내 재-턴오버/인터셉트/볼 로스트 빈도
   - 패스 성공률 급락 구간
   - 짧은 시간에 실패 이벤트가 몰리는 패턴

5. **세트피스/리스타트 컨텍스트**
   - 프리킥/코너/스로인 직후 시퀀스

6. **경기 상태 컨텍스트**
   - 시간대(전/후반, 종료 임박)
   - 점수차
   - 직전 1분 이벤트 밀도(tempo)

### 3-3. 외부 데이터 없이 성능 개선 방법

- **시퀀스 피처 강화**: 최근 5초/10초/20초 윈도우 집계 다층화
- **라벨 정의 재점검**: 
  - 중복 양성 처리
  - negative sampling 완화
- **랭킹 목적함수/후처리**: Precision@10 목표면 랭킹 손실/후처리 활용

## 4. 전술 분석 최소 데이터 요구사항

| 전술 정보 | Track2만으로? | 최소 추가 데이터 |
|---------|------------|--------------|
| 오프사이드 판정(라인) | 사실상 불가 | 공격/수비 선수 위치 + 공 위치 (패스 순간) |
| 압박 강도(정량) | 약한 프록시만 가능 | 추적(선수/공 좌표) 또는 이벤트 기반 Pressure |
| 선수 간 거리/포메이션 | 불가 | 선수 위치 추적 + 팀/선수 ID |
| 박스 내 공간 점유 | 불가 | 선수 추적(박스 내부 인원 카운트) |
| 전술 패턴 라벨 학습 | 제한적 | AIHub처럼 전술 라벨 포함 데이터 |

**결론**: Track2만으로도 "공의 이동 + 이벤트 시퀀스" 기반 슛 예측은 가능하지만, 오프사이드 라인/진짜 압박/공간 점유는 추적 없이는 원천적으로 불가.

## 5. PR-AUC / Precision@10 개선 전략

### 전략 1) 이벤트 로그 고도화 (피처 엔지니어링/시퀀스화)
- **난이도**: 중
- **기대효과**: 가장 확실하게 "지금 당장" 올릴 수 있음
- **추천**: 외부 데이터 제약이 있는 대회 환경에서 1순위

### 전략 2) 외부 데이터 통합: AIHub 영상→이벤트 로그 변환
- **난이도**: 매우 높음
- **기대효과**: 성공하면 크지만, 실패 리스크/시간 리스크 큼
- **주의**: 대회 규정 확인 필요 (외부 데이터/영상 사용 제한 가능)

### 전략 3) 하이브리드: 이벤트 로그 + 최소 위치(선수/공 좌표만)
- **난이도**: 높음
- **기대효과**: 압박/공간 피처가 열리면 Precision@k 개선
- **전제**: 정말로 좌표가 있어야 함 (SkillCorner/Bundesliga 등)

### 전략 4) 전이학습: 다른 축구 이벤트 데이터로 사전학습
- **난이도**: 중~높음
- **기대효과**: Track2가 작은 데이터면 도움이 될 수 있음
- **추천 데이터**: Wyscout 공개 이벤트 데이터 (CC BY)

### 우선순위
- **대회 제출 중심**: (1) → (4) → (3) → (2)
- **R&D/데모 중심**: (3) 또는 (2)가 상한을 더 밀어올릴 수 있음

## 6. AIHub 통합 로드맵

### Step 0. "쓸 수 있는지" 확정
1. AIHub 71482 다운로드
2. 샘플 JSON 1개 파싱하여 확인:
   - 전술 라벨이 "클립 단위"인지
   - 추적/좌표가 "프레임 단위"로 있는지
   - 타임스탬프가 실제 경기 시간과 매칭되는지

### Step 1. 데이터 변환 파이프라인
- **(A) 추적/라벨이 충분히 있음**: JSON 파서/정규화 → 피치 좌표화 → 압박/간격/점유 파생 피처
- **(B) bbox만 있고 피치 좌표 없음**: 카메라 캘리브레이션/호모그래피 필요 (난이도 급상승)

### Step 2. 라벨 활용
- 전술 라벨 예측 모델 (보조 태스크)
- 전술 라벨을 will_have_shot의 입력 피처로 사용
- 멀티태스크/지식증류

### Step 3. 앙상블
- 이벤트 로그 모델: 대회 제출의 안정성
- 영상/추적 모델: 데모/확장성
- Late-fusion (점수 결합)
- **주의**: 대회 규정상 제출 가능 여부 확인 필수
